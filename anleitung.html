Für ein gemeinsames Software-Projekt nutzen wir Git und GitHub

Installation und Setup

1. Erstelle einen kostenlosen account bei GitHub. Die Einrichtung muss über eine gültige email-Adresse bestätigt werden. Benutzername und email-Adresse werden später bei der Einrichtung von Git benötigt.

2. Installiere die Software Git. Die default-Einstellungen können beibehalten werden.

3. Clicke auf Git Bash, es öffnet sich eine Konsole. Mache folgende Eingaben mit dem Benutzernamen und der email-Adresse deines GitHub-accounts. 

$ git config --global user.name "malteRiedberg"
$ git config --global user.email "malte.riedberg@googlemail.com"

4. Installiere die Software GitHub Desktop. Clicke auf Sign into Github. Gib anschließend noch einen Namen ein, über den ich dich identifizieren kann (sollte also ein bisschen Ähnlichkeit mit deinem richtigen Namen haben).

Installation und Setup

1. Melde dich bei GitHub an. Gib im Suchfeld oben links ein: ktheu/sometools ein, klicke dann auf den Namen des gefundenen Repositories und dann oben rechts auf den button fork. Dadurch wird eine Kopie des Repos ktheu/sometools in deinem GitHub account erstellt: malteRiedberg/sometools.

2. Starte GitHub Desktop und clicke auf clone a repository . Selektiere das gerade auf GitHub erzeugte (remote) Repository. Passe den Local path an und clicke auf clone . Jetzt sind die Files in deinem lokalen repository.

Installation und Setup

1. Füge ein den sometools ein weiteres Feature hinzu.

2. Öffne GitHub Desktop. Die geänderten Files werden angezeigt. Schreibe in das Feld summary einen kurzen Text, z.B. add featurename

und klicke auf Commit to master . Damit sind die Änderungen in deinem lokalen Repository gespeichert. Dann klicke oben auf Push origin . Dadurch werden die Änderungen in dein remote Repository auf GitHub übertragen.

3. Gehe nach GitHub und klicke auf New pull request . Es kommt hoffentlich die Meldung Able to merge . Klicke auf Create pull request, schreibe ggf. noch weitere Erläuterungen und klicke dann nochmal auf Create pull request

Update der eigenen Repositories

1. Wenn deine Änderung akzeptiert wurde, bekommst du in GitHub die Meldung, dass dein commit in das Repository ktheu/master gemerged wurde.

2. Wenn in ktheu/master weitere Änderungen gemacht wurden, bekommst du in GitHub eine Meldung wie this branch is 1 commit behind ktheu/master. Um das eigene remote repo mit den Ursprung des forks zu synchronisieren, muss man zunächst das eigene lokale Repo synchronisieren. Öffne GitHub Desktop und wähle Repository - Open in Command Prompt . Gib dort den Befehl ein: git fetch upstream. Dadurch wird im lokale Repo ein branch mit dem neuen Ursprungs-Repo erstellt. Selektiere dann Branch - merge into current branch . Klicke anschließend Push origin um den neuen Stand auf den dein remote Repo zu übertragen.

Anforderungen an neue Features am Beispiel collatz

Das Feature steht in der Datei collatz.py. In dieser Datei steht die eigentliche Berechnung in einer Funktion def collatz. Das UserInterface, d.h. alle Ein- und Ausgaben stehen in def collatz_ui. In dem File sometools.py muss der Name des Features und ggf. eine Abkürzung eingetragen werden.
